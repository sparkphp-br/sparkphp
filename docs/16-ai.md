# AI SDK

O SparkPHP trata AI como parte do runtime do framework, nao como um pacote colado por fora.
A entrada continua curta:

```php
ai()
```

Mas a superficie publica cobre texto, embeddings, imagem, audio, agentes e retrieval
com a mesma linguagem de builders pequenos e previsiveis do restante do core.

## Filosofia

- uma entrada unica: `ai()`
- mesmos principios do framework: pouco wiring, boa convencao e feedback rapido
- driver `fake` por default para desenvolvimento, suite e smoke tests
- providers reais plugados via `AiProvider`, sem mudar a API da aplicacao
- observabilidade first-party no Inspector e no CLI

## Configuracao

No `.env`:

```env
AI_DRIVER=fake
AI_TEXT_MODEL=spark-text
AI_EMBEDDING_MODEL=spark-embedding
AI_IMAGE_MODEL=spark-image
AI_IMAGE_SIZE=1024x1024
AI_AUDIO_MODEL=spark-audio
AI_AUDIO_VOICE=default
AI_AUDIO_FORMAT=mp3
AI_AGENT_MODEL=spark-agent

SPARK_AI_MASK=true
SPARK_AI_TRACE_PREVIEW=240
```

Os dois ultimos valores controlam o trace de AI no Spark Inspector:

- `SPARK_AI_MASK=true`: mascara prompts, respostas, contexto e tool results
- `SPARK_AI_TRACE_PREVIEW=240`: limita previews longos no painel interno

Para diagnosticar o estado atual do runtime:

```bash
php spark ai:status
```

Para um smoke test do provider:

```bash
php spark ai:smoke-test
php spark ai:smoke-test --capability=retrieval
```

## Mapa rapido da SDK

```php
ai()->text(...)
ai()->embeddings(...)
ai()->image(...)
ai()->audio(...)
ai()->agent(...)
ai()->retrieve(...)
```

Todos esses builders seguem o mesmo padrao:

1. configurar input, modelo e opcoes
2. chamar `generate()`, `run()` ou `get()`
3. consumir um response tipado, com `provider`, `model` e `meta`

## Texto

### Exemplo basico

```php
$response = ai()
    ->text('Resuma a proposta comercial em 3 bullets.')
    ->system('Seja objetivo.')
    ->temperature(0.2)
    ->maxTokens(300)
    ->generate();

return (string) $response;
```

### O que o response entrega

```php
$response->text;
$response->provider;
$response->model;
$response->structured; // null quando nao ha schema
$response->meta;
```

O `meta` pode trazer:

- `usage.input`
- `usage.output`
- `usage.total`
- `cost_usd`
- detalhes especificos do provider

Exemplo:

```php
$tokens = $response->meta['usage']['total'] ?? 0;
$cost = $response->meta['cost_usd'] ?? 0.0;
```

## Structured output

Texto e agentes aceitam `schema([...])` para pedir payload estruturado.

```php
$result = ai()->text('Extraia nome e email')
    ->schema([
        'type' => 'object',
        'properties' => [
            'name' => ['type' => 'string'],
            'email' => ['type' => 'string', 'format' => 'email'],
            'qualified' => ['type' => 'boolean'],
        ],
    ])
    ->generate();

return $result->structured;
```

No driver `fake`, o Spark gera uma estrutura deterministica. Isso e util para:

- contratos de API
- testes locais
- smoke tests repetiveis

## Embeddings

```php
$embeddings = ai()
    ->embeddings([
        'SparkPHP prioriza previsibilidade.',
        'Laravel prioriza ecossistema.',
    ])
    ->generate();

$firstVector = $embeddings->first();
```

O response inclui:

- `vectors`
- `provider`
- `model`
- `meta.dimensions`
- `meta.usage`
- `meta.cost_usd`

Embeddings sao a base para:

- semantic search
- retrieval
- recomendacao por similaridade
- RAG curto

## Agentes

```php
$agent = ai()
    ->agent('support')
    ->instructions('Responda de forma curta e tecnica.')
    ->tool('lookupOrder', fn(array $args) => ['id' => $args['id'] ?? null], 'Consulta pedido')
    ->context(['team' => 'support'])
    ->prompt('Como encontro o pedido 123?')
    ->run();

return (string) $agent;
```

O response do agente inclui:

- `text`
- `tools`
- `context`
- `structured`
- `meta.agent`
- `meta.tool_calls`
- `meta.tool_results`
- `meta.usage`
- `meta.cost_usd`

### Quando usar `agent()`

Use `agent()` quando a operacao precisar de uma ou mais destas coisas:

- instrucoes separadas do prompt
- tools
- contexto nomeado
- structured output
- passos controlados por `maxSteps()`

Se o caso for so completar texto, `text()` costuma ser o ponto mais simples.

## Retrieval

O mesmo cliente sobe retrieval sem glue code paralelo:

```php
$documents = ai()->retrieve('Como configuro cache?')
    ->from('documents', 'embedding')
    ->select('id', 'title', 'content')
    ->take(3)
    ->get();

$context = $documents->toPromptContext('content');
```

`AiRetrievalResult` inclui:

- `items`
- `provider`
- `model`
- `meta.metric`
- `meta.vector_column`
- `meta.limit`
- `meta.result_count`
- `meta.vector_dimensions`
- `meta.usage`
- `meta.cost_usd`

Ele tambem expone:

```php
$documents->first();
$documents->toArray();
$documents->toPromptContext('content');
```

Para o guia detalhado de semantic search e retrieval, veja
[18-search.md](18-search.md).

## Fluxo tipico de RAG curto

```php
$retrieval = ai()->retrieve('Como configuro cache?')
    ->from(Article::class, 'embedding')
    ->take(3)
    ->get();

$answer = ai()->agent('support')
    ->prompt(
        "Contexto:\n"
        . $retrieval->toPromptContext('content')
        . "\n\nPergunta: Como configuro cache?"
    )
    ->run();
```

## Driver fake

O driver `fake` e o comportamento default da SDK no estado atual do framework.

Exemplo:

```php
$text = ai()->text('Hello')->generate();

echo (string) $text;
// Fake response: Hello
```

Ele existe para resolver tres problemas muito reais:

- desenvolver sem credenciais
- testar sem custo e sem rede
- garantir respostas deterministicas para a suite

O fake provider agora tambem publica `usage` e `cost_usd`, o que ajuda a validar
telemetria e observabilidade localmente.

## Respostas tipadas

As classes publicas da SDK sao:

- `AiManager`
- `AiClient`
- `AiProvider`
- `AiTool`
- `AiTextRequest`, `AiEmbeddingRequest`, `AiImageRequest`, `AiAudioRequest`, `AiAgentRequest`
- `AiTextResponse`, `AiEmbeddingResponse`, `AiImageResponse`, `AiAudioResponse`, `AiAgentResponse`
- `AiRetrievalResult`
- `AiFakeProvider`

Em codigo de aplicacao, o normal e interagir com os builders e os responses, sem
precisar conhecer os detalhes internos do provider.

## Registrando um provider custom

```php
$manager = app()->getContainer()->make(AiManager::class);

$manager->extend('custom', function () {
    return new class implements AiProvider {
        public function name(): string { return 'custom'; }

        public function text(AiTextRequest $request): AiTextResponse
        {
            return new AiTextResponse(
                'ok',
                'custom',
                $request->model,
                meta: ['usage' => ['input' => 4, 'output' => 2, 'total' => 6]]
            );
        }

        public function embeddings(AiEmbeddingRequest $request): AiEmbeddingResponse
        {
            return new AiEmbeddingResponse([[1.0, 0.0]], 'custom', $request->model);
        }

        public function image(AiImageRequest $request): AiImageResponse
        {
            return new AiImageResponse('image', 'custom', $request->model);
        }

        public function audio(AiAudioRequest $request): AiAudioResponse
        {
            return new AiAudioResponse('audio', 'custom', $request->model);
        }

        public function agent(AiAgentRequest $request): AiAgentResponse
        {
            return new AiAgentResponse('agent', 'custom', $request->model);
        }
    };
});

$reply = ai('custom')->text('ping')->generate();
```

## Diagnostico e operacao

### Status atual do runtime

```bash
php spark ai:status
php spark ai:status --json
```

Esse comando mostra:

- driver ativo
- provider resolvido
- modelos por capacidade
- tamanho default de imagem
- voz/formato default de audio
- configuracao de trace no Inspector

### Smoke test

```bash
php spark ai:smoke-test
php spark ai:smoke-test --capability=text
php spark ai:smoke-test --capability=retrieval --json
```

Por default, o smoke test cobre texto, embeddings, imagem, audio e agente. Retrieval
fica como capacidade explicita porque depende da camada de dados estar acessivel.

O comando retorna latencia, tokens, custo e um resumo curto por capacidade.

## Observabilidade

Quando o Spark Inspector esta ativo, chamadas de AI passam a registrar automaticamente:

- tipo da operacao (`text`, `embeddings`, `image`, `audio`, `agent`, `retrieval`)
- provider e model
- latencia
- tokens
- custo estimado
- tool calls
- previews de request/response

Para a documentacao dedicada do painel, mascaramento e leitura das metricas, veja
[19-ai-observability.md](19-ai-observability.md).
