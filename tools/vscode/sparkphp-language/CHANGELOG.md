# Changelog

## 0.4.0

- added active VS Code runtime with directive completion provider
- added pipe completion provider for Spark echoes
- added hover help for directives and pipes

## 0.3.0

- added more snippets for common Spark directives
- improved directive tokenization for editor themes
- added VS Code launch/task files for local extension development

## 0.2.0

- improved syntax scopes for directives, pipes, named arguments and constants
- better Blade-like highlighting feel for `.spark` files in VS Code
- refreshed local VSIX package

## 0.1.0

- initial SparkPHP `.spark` language support
- TextMate grammar for Spark directives and echoes
- snippets for common template patterns
- editor configuration for folding and auto-closing pairs
